# PPT 口播稿生成

为指定的 PPT 生成逐页口播稿文本，并可选生成 TTS 音频。

## 触发条件

当用户要求为某个 PPT 生成口播稿、演讲稿、讲解稿时触发本技能。

## 执行流程

### Step 0: 确认目标 PPT

检查 `runtime.state` 中的 `current_ppt_task_id`：

**如果非空** → 已明确目标 PPT，直接进入 Step 1。

**如果为空** → 用户未指定具体 PPT，需要从系统提示的“当前PPT产出”表格中获取可用列表：

1. **系统提示中没有 PPT** → 告知用户：“当前工作区没有已完成的 PPT，请先使用 /ppt 生成。”
2. **只有 1 个 PPT** → 直接使用该 PPT 的 task_id，告知用户：“将为《XX》生成口播稿。”然后进入 Step 1。
3. **有多个 PPT** → 调用 `clarify_form` 让用户选择：
   ```
   clarify_form(
     title="请选择要生成口播稿的 PPT",
     fields=[{
       name: "selected_ppt_task_id",
       label: "目标PPT",
       type: "select",
       options: ["PPT标题1", "PPT标题2", ...]
     }]
   )
   ```
   用户选择后，根据标题匹配对应的 task_id，然后进入 Step 1。

### Step 1: 获取 PPT 大纲

使用 Step 0 确定的 task_id（从 `current_ppt_task_id` 或用户选择中获取），调用 `get_ppt_detail(task_id=...)` 获取 PPT 的结构化信息，包括：
- 幻灯片标题、核心要点、关键词
- 视觉风格、目标受众等元信息

**如果大纲不存在**（工具返回提示"未保存结构化大纲"）：
- 告知用户："该 PPT 未保存大纲信息，我将直接基于幻灯片内容生成口播稿，质量可能略有下降。"
- 继续后续步骤，但跳过 RAG 检索（因为没有关键词）

### Step 2: RAG 增强检索

对于大纲中**每张幻灯片**的 `keywords` 字段：
1. 将关键词组合为查询语句（如："消防安全 新员工入职 安全意识"）
2. 调用 `rag_search(query=关键词组合, top_k=3)` 获取知识库原文
3. 记录检索结果，用于 Step 3 的口播稿生成

**如果 RAG 无结果**：
- 标注该页"无参考资料"，仅基于大纲的 `key_points` 生成口播稿
- 不影响其他页面的检索

**如果知识库为空**（文档已被删除）：
- 告知用户："部分原始培训文档已移除，口播稿将主要基于大纲内容生成。"
- 继续生成，但降低对原文引用的依赖

### Step 3: 生成口播稿

基于以下信息为每张幻灯片撰写口播稿：
- 大纲中的 `key_points`（骨架）
- RAG 检索到的原文段落（深度素材）
- 用户的定制化要求（如果消息中包含）

#### 写作规范

**语言**：
- 默认使用中文
- 如果用户明确要求英文，则切换为英文
- **绝不中英文混用**，保持语言一致

**风格**：
- 口语化、自然流畅，像真人讲师在台上讲解
- 避免朗读式复述 PPT 文字，要展开讲解
- 使用过渡语衔接各页：
  - "接下来我们来看看..."
  - "了解了 XX 之后，我们再来看..."
  - "这里有一个关键点..."
  - "请大家注意..."

**结构**：
- 第 1 页（封面）：包含开场白、自我介绍（如有）、课程导入
- 中间页：逐页展开讲解
- 最后一页：总结要点、后续建议、结束语

**篇幅**：
- 每页口播稿控制在 **100-250 字**
- 重点页面可适当增加，但不超过 350 字

**输出格式**（在脑海中组织，最终传给 save_narration）：

```
【第1页：封面】
各位同事大家好，欢迎来到今天的培训...

【第2页：火灾的基本知识】
说到火灾，我们首先要了解它的成因...

【第3页：灭火器使用方法】
了解了火灾的原理之后，我们来看如何灭火...
```

### Step 4: 保存口播稿

生成完所有页面的口播稿后，调用 `save_narration` 工具：

```
save_narration(
  parent_task_id=<Step 0 确定的 task_id>,
  title="<PPT标题> · 口播稿",
  slides=<JSON string>,
  language="zh"
)
```

`slides` JSON 格式：
```json
[
  {"number": 1, "title": "封面", "text": "各位同事大家好..."},
  {"number": 2, "title": "火灾的基本知识", "text": "说到火灾..."},
  {"number": 3, "title": "灭火器使用方法", "text": "了解了火灾..."}
]
```

**注意**：`slides` 必须是完整的 JSON 字符串，包含所有幻灯片的口播稿文本。

### Step 5: 确认反馈

口播稿保存后，告知用户：
- 口播稿已保存，可在右侧产出面板查看
- TTS 音频正在后台生成（如果 TTS 已配置）
- 如何查看进度和下载

## 定制化需求处理

如果用户在触发消息中附带了额外要求（如"语气轻松一些"、"重点强调第三章"、"控制在10分钟以内"），在生成口播稿时融入这些要求：

- 语气要求 → 调整用词和句式
- 重点要求 → 增加对应页面的篇幅
- 时长要求 → 调整每页字数（约 200 字/分钟）

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 无可用 PPT | 引导用户先生成 PPT |
| 大纲缺失 | 降级为基于 HTML 内容生成 |
| RAG 无结果 | 仅基于大纲要点生成，标注无参考 |
| 文档已删除 | 告知用户，继续基于大纲生成 |
| save_narration 调用失败 | 将口播稿文本直接展示给用户作为备选 |
