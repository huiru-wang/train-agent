# 桃紫分境

## 1. Vibe — 整体气质

柔和、明快、有编辑感。以温暖的蜜桃色与清冷的薰衣草紫形成左右分屏，像翻开一本装订精美的培训手册。糖果色的标签与半透明卡片让严肃的规约内容变得轻盈易读，整体气质介于「知识手账」与「轻培训课件」之间。

关键词：分屏 pastel / 糖果标签 / 大号编号 / 半透明卡片 / 轻弹跳动画

## 2. Color System — 色彩系统

```css
:root {
    --bg-primary: #e4dff0;      /* 薰衣草紫：右侧主内容区 */
    --bg-secondary: #f5e6dc;    /* 蜜桃色：左侧标题区 */
    --text-primary: #1a1a1a;    /* 近黑：主标题与正文 */
    --text-secondary: #5a5a6a;  /* 中灰：副标题与说明 */
    --accent-main: #b8a0d8;     /* 薰衣草强调：进度条、导航点、网格 */
    --accent-highlight: #f0c8a0;/* 蜜桃强调：渐变、高亮词底 */
    --accent-muted: rgba(180, 160, 210, 0.15); /* 弱化网格线 */
    --surface: rgba(255, 255, 255, 0.55);      /* 卡片表面 */
    --badge-mint: #c8f0d8;
    --badge-yellow: #f0f0c8;
    --badge-pink: #f0d4e0;
}
```

| 变量 | 色值 | 角色 | 使用场景 |
|------|------|------|----------|
| `--bg-primary` | `#e4dff0` | 薰衣草紫 | 页面主背景、右侧内容区 |
| `--bg-secondary` | `#f5e6dc` | 蜜桃色 | 左侧分栏、标题区背景 |
| `--text-primary` | `#1a1a1a` | 近黑 | 主标题、正文、编号 |
| `--text-secondary` | `#5a5a6a` | 中灰 | 副标题、说明、页脚 |
| `--accent-main` | `#b8a0d8` | 紫强调 | 进度条、导航点、装饰网格 |
| `--accent-highlight` | `#f0c8a0` | 桃强调 | 渐变高亮、关键词底色 |
| `--accent-muted` | `rgba(180,160,210,0.15)` | 弱化紫 | 背景网格纹理 |
| `--surface` | `rgba(255,255,255,0.55)` | 卡片表面 | 规则卡片、列表项背景 |
| `--badge-mint` | `#c8f0d8` | 薄荷标签 | 推荐/正向标签 |
| `--badge-yellow` | `#f0f0c8` | 柠黄标签 | 参考/注意标签 |
| `--badge-pink` | `#f0d4e0` | 柔粉标签 | 强制/关键标签 |

**配色逻辑**：左右分屏的冷暖 pastel 对比是风格核心；信息层级通过白底卡片与糖果色标签区分，不额外引入高饱和色，保持整体温柔但清晰的辨识度。

## 3. Typography — 字体系统

| 用途 | 中文字体 | 西文字体 | 字重 |
|------|----------|----------|------|
| 超大标题 / 封面主标题 | Noto Sans SC | Outfit | ExtraBold / 800 |
| 页面主标题 / 章节标题 | Noto Sans SC | Outfit | Bold / 700 |
| 副标题 / 引导段 | Noto Sans SC | Outfit | Medium / 500 |
| 正文 / 卡片说明 | Noto Sans SC | Outfit | Regular / 400 |
| 标签 / 编号 | Noto Sans SC | Outfit | Bold / 700 |
| 巨型编号（水印） | — | Outfit | ExtraBold / 800 |

**导入示例**：

```
https://fonts.loli.net/css2?family=Outfit:wght@400;500;700;800&family=Noto+Sans+SC:wght@400;500;700;900&display=swap
```

**排版规则**：

- 封面主标题：几何无衬线 ExtraBold，视口宽度 5.5–8vw，紧凑行高 1.05，左对齐
- 英文副标题 / 引导小字：Outfit Medium，小字号，颜色取 `--text-secondary`
- 页面主标题：Outfit Bold，2.2–3.2vw，左对齐
- 正文：Noto Sans SC / Outfit Regular，0.75–1.05vw，行高 1.4–1.5
- 标签文字：Outfit Medium / Bold，11–13px，使用深色文字与糖果色底形成高对比
- 巨型编号：Outfit ExtraBold，6–8vw，透明度 5%，作为左侧分栏的水印装饰

**核心原则**：几何无衬线 Outfit 负责整体现代感与可读性，中文 fallback 使用思源黑体，保证中西文混排时的清晰对齐。

## 4. Layout Grammar — 布局语法

整体采用左右分屏布局，左侧约占 38–45%，右侧约占 55–62%。所有页面全屏显示，四周保留响应式安全边距。

### Cover / 封面页

出现频次：1 页

**结构**：左右分屏。左侧蜜桃色分栏底部对齐，包含引导标签、超大主标题、副标题说明；右侧薰衣草紫分栏顶部为横向标签组，中部为口号式金句，底部为模块说明。

**背景**：左侧 `--bg-secondary` 纯色，右侧 `--bg-primary` 纯色叠加淡紫色网格纹理。

**占位符**：`引导标签`、`封面主标题`、`封面副标题`、`模块标签 1–N`、`封面金句`、`模块说明`

**文字颜色**：主标题与金句用 `--text-primary`，引导标签与说明用 `--text-secondary`，标签组文字随糖果色底使用深色文字。

---

### Content / 内容页 — 规则卡片型

出现频次：4–5 页

**结构**：左侧分栏顶部放置超大半透明编号水印，下方为章节编号、中文主标题、英文副标题。右侧分栏使用 2×2 或 2×3 的规则卡片网格，每张卡片包含标签（强制/推荐/参考）、规则文本、代码示例。

**背景**：左侧蜜桃色，右侧薰衣草紫 + 淡网格。

**占位符**：`章节编号`、`页面主标题`、`页面英文副标题`、`卡片 1 标签`、`卡片 1 规则文本`、`卡片 1 示例`（共 4–6 组）

**文字颜色**：标题与规则文本用 `--text-primary`，示例与副标题用 `--text-secondary`，标签文字使用与底色对应的深色（如粉色底配 `#a04060`）。

---

### Content / 内容页 — 规则列表型

出现频次：2–3 页

**结构**：左侧分栏与卡片型相同。右侧分栏改为纵向堆叠的规则条目列表，每个条目包含圆形图标、规则文本、代码示例。

**背景**：与卡片型一致。

**占位符**：`章节编号`、`页面主标题`、`页面英文副标题`、`列表项 1 图标`、`列表项 1 文本`、`列表项 1 示例`（共 4–6 组）

**文字颜色**：同规则卡片型。

---

### Closing / 结尾页

出现频次：1 页

**结构**：左右分屏，但左侧比例略小（约 40%）且文字右对齐。左侧包含英文引导小字与宣言式大标题；右侧包含说明文字、横向标签组、行动按钮。

**背景**：左侧蜜桃色，右侧薰衣草紫 + 淡网格。

**占位符**：`结尾引导小字`、`结尾宣言主标题`、`结尾说明`、`标签组`、`行动按钮文本`

**文字颜色**：宣言大标题用 `--text-primary`，说明用 `--text-secondary`，按钮为白底深色文字。

## 5. Signature Elements — 标志性元素

1. **左右 pastel 分屏**：所有页面固定左侧蜜桃色 / 右侧薰衣草紫，形成强烈的系列识别度。
2. **淡紫色网格纹理**：右侧背景叠加 40px 间距的淡紫网格线，透明度 12%，营造 subtle 的结构感。
3. **巨型编号水印**：左侧分栏顶部放置 01、02、03 等超大半透明编号，既是章节标识也是装饰。
4. **糖果色药丸标签**：薄荷绿、柠黄、柔粉三色圆角标签，分别对应推荐、参考、强制等信息角色。
5. **半透明规则卡片**：右侧内容区使用 `rgba(255,255,255,0.55)` 的白底卡片，hover 时提升至 0.85 并微弹。
6. **进度条与导航点**：顶部固定 4px 渐变进度条（蜜桃到薰衣草），右侧固定垂直导航点。
7. **轻弹跳入场动画**：元素使用 `cubic-bezier(0.34, 1.56, 0.64, 1)` 的弹性缓动，从下方 20px 淡入。
8. **圆角几何**：卡片与标签使用 12–20px 圆角，与硬朗的分屏直线形成柔化对比。

## 6. Visual Assets — 视觉资产

本风格不依赖外部图片资源，所有视觉效果由 CSS 生成。图标使用简洁的 emoji 或开源线性图标库（如 Lucide），颜色跟随当前标签底色或文字色。

若后续需要加入图片：

- 图片容器应使用圆角矩形，保持与卡片一致的视觉语言；
- 图片色调建议偏暖 pastel，与整体配色协调；
- 图片区域需避开左侧主标题与右侧卡片文字的安全区。

## 7. Usage Guidelines — 生成新 PPT 时的使用说明

1. **页面类型选择**：新 PPT 必须包含封面页、若干内容页（卡片型或列表型择一或混用）、结尾页。
2. **配色使用**：严格使用 pastel 分屏配色，不要引入新的高饱和强调色；信息层级通过卡片、标签、字号区分。
3. **字体限制**：标题与正文均使用 Outfit + Noto Sans SC fallback，禁止使用 Calibri / Arial / Microsoft YaHei 等系统默认字体。
4. **分屏比例**：左侧 38–45%，右侧 55–62%，根据内容密度微调；封面左侧可略大，结尾页左侧可略小。
5. **标签语义**：强制类用粉色底、推荐类用薄荷绿底、参考类用柠黄底，标签文字始终使用深色以保证可读性。
6. **编号系统**：内容页按 01、02、03 顺序编号，编号水印透明度控制在 5% 左右。
7. **动画节奏**：保持轻弹入场动画，但减少运动偏好用户的负担；复杂页面元素较多时可适当减少动画延迟。
8. **背景处理**：始终使用 CSS 生成的纯色 + 网格纹理，不要替换为写实摄影背景。
