# 墨纸杂志

## 1. Vibe — 整体气质

克制、沉浸、有编辑感。以高对比的「墨黑 + 纸白」建立纸媒质感，以衬线大字作为视觉重音，以等宽小字作为节奏装饰。整体气质介于「独立杂志」与「学术期刊」之间——既有大段文字的阅读信任感，又有大标题、金句页、数据页的冲击力。

关键词：纸媒感 / 高对比 / 衬线重音 / 杂志页眉 / 强留白

## 2. Color System — 色彩系统

本风格提供 5 套预设主题，**必须整套使用，不允许自定义 hex 或混搭**。

### 默认主题：墨水经典（Monocle）

```css
:root {
    --bg-primary: #f1efea;
    --bg-secondary: #e8e5de;
    --text-primary: #0a0a0b;
    --text-secondary: #5a5a58;
    --accent-main: #0a0a0b;
    --accent-highlight: #0a0a0b;
    --accent-muted: #d8d5ce;
    --surface: #e8e5de;
}
```

| 变量 | 色值 | 角色 | 使用场景 |
|------|------|------|----------|
| `--bg-primary` | #f1efea | 纸白 | 浅色页背景、深色页文字 |
| `--bg-secondary` | #e8e5de | 纸灰 | 卡片底色、引用框背景、hover 状态 |
| `--text-primary` | #0a0a0b | 墨黑 | 标题、正文主色 |
| `--text-secondary` | #5a5a58 | 中灰 | 辅助说明、英文副标题、弱化文字 |
| `--accent-main` | #0a0a0b | 主强调 | 页眉页脚、编号、线条、按钮 |
| `--accent-highlight` | #0a0a0b | 高亮强调 | 关键声明、重点数字、金句 |
| `--accent-muted` | #d8d5ce | 柔化标记 | 标签底色、细线、hover 底色 |
| `--surface` | #e8e5de | 卡片表面 | 数据卡片、引用框、信息区块 |

### 其他可选主题

| 主题 | 调性 | 适用场景 |
|------|------|----------|
| 靛蓝瓷（Indigo Porcelain） | 深靛蓝 + 瓷白 | 科技、研究、数据、工程师文化 |
| 森林墨（Forest Ink） | 深森林绿 + 象牙 | 自然、可持续、文化、非虚构 |
| 牛皮纸（Kraft Paper） | 深棕 + 暖米 | 怀旧、人文、阅读、独立杂志 |
| 沙丘（Dune） | 炭灰 + 沙色 | 艺术、设计、创意、审美分享 |

**配色逻辑**：每份 deck 仅使用一套主题。色彩以两极对比为主，不引入第三色做装饰。强调信息通过字号、字重、留白和衬线/无衬线对比完成，而非依赖彩色块。

## 3. Typography — 字体系统

| 用途 | 中文字体 | 西文字体 | 字重 |
|------|----------|----------|------|
| 超大标题 / 金句 | Noto Serif SC | Playfair Display | Bold / 700 |
| 页面主标题 | Noto Serif SC | Playfair Display | SemiBold / 600 |
| 副标题 / 引导段 | Noto Serif SC | Source Serif 4 | Regular / 400 |
| 正文 / 说明 | Noto Sans SC | IBM Plex Sans | Regular / 400 |
| 页眉 / 页脚 / 标签 | — | IBM Plex Mono | Medium / 500 |
| 数字 / KPI | Noto Serif SC | Playfair Display | ExtraBold / 800 |

**导入示例**：

```
https://fonts.loli.net/css2?family=Playfair+Display:ital,wght@0,400;0,500;0,600;0,700;0,800;0,900;1,400;1,700&family=Source+Serif+4:ital,opsz,wght@0,8..60,300;0,8..60,400;0,8..60,500;0,8..60,600;1,8..60,400&family=IBM+Plex+Mono:wght@300;400;500;600&family=Noto+Serif+SC:wght@300;400;500;600;700;900&family=Noto+Sans+SC:wght@300;400;500;700;900&family=IBM+Plex+Sans:wght@400;500;700&display=swap
```

**排版规则**：
- 页面主标题：衬线，视口宽度 7–11vw，左对齐或居中（仅 hero/金句页），字间距紧缩
- 英文副标题 / 引导小字：全大写 Mono，12–14px，字间距 +0.18em，颜色取 `--text-secondary`
- 正文：无衬线，14–18px（随 viewport 可放大至 1.15vw），行高 1.6–1.75
- 数字：Playfair Display，用于数据页与 KPI，字间距 -0.03em
- 引用：衬线加粗，视口宽度 5–6vw，仅用于大引用页

**核心原则**：衬线用于「视觉重音」（标题、金句、数字），无衬线用于「信息密度」（正文、列表、说明），Mono 用于「装饰节奏」（页眉页脚、标签、编号）。

## 4. Layout Grammar — 布局语法

### Cover / 封面页

出现频次：1 页

**结构**：页面顶部为杂志页眉，左侧为栏目名，右侧为卷号 / 页码。中央垂直居中排列：引导小字 → 超大主标题 → 副标题 → 引导段 → 元数据行。底部为杂志页脚，左侧为说明文字，右侧为年份。

**背景**：深色背景（hero dark），可使用动态网格或流体渐变，让背景大面积透出以营造仪式感。

**占位符**：`栏目名`、`卷号`、`引导小字`、`封面主标题`、`封面副标题`、`引导段`、`元数据`、`页脚说明`、`年份`

**文字颜色**：主标题 / 副标题用 `--bg-primary`，引导小字 / 元数据 / 页脚用 `--bg-primary` 60–70% 透明度。

---

### Section / 章节过渡页

出现频次：1–2 页

**结构**：与封面类似，但内容更极简。中央垂直居中：引导小字（Act I / Act II）→ 大号章节标题 → 一行引语。页眉写「第 N 幕 · 章节名」和页码。

**背景**：hero light 与 hero dark 必须交替出现，制造呼吸节奏。

**占位符**：`幕号`、`章节标题`、`引语`、`页码`

**文字颜色**：hero light 页用 `--text-primary` 为主，hero dark 页用 `--bg-primary` 为主。

---

### Content / 内容页 — 数据大字报型

出现频次：1 页

**结构**：顶部页眉 + 页码。主体为引导小字 → 中号标题 → 引导段 → 3×2 或 2×2 数据卡片网格。每张卡片结构固定：顶部 Mono 英文标签 → 巨型数字 → 底部中文注释。

**背景**：`--bg-primary` 纯色。

**占位符**：`引导小字`、`页面主标题`、`引导段`、`标签 1`、`数字 1`、`注释 1`（共 4–6 组）

**文字颜色**：标签与页脚用 `--text-secondary`，数字用 `--text-primary`，注释用 `--text-secondary`。

---

### Content / 内容页 — 左文右图型

出现频次：1–2 页

**结构**：页眉 + 页码。主体为 7:5 两栏网格，左栏为引导小字 → 大标题 → 引导段 → 底部引用框；右栏为一张 16:10 或 4:3 主图。左栏使用 flex 纵向布局并让引用框自然贴底。

**背景**：`--bg-primary` 或深色背景，light 与 dark 交替。

**占位符**：`引导小字`、`大标题`、`引导段`、`引用文字`、`引用出处`、`图片说明`

**文字颜色**：标题用 `--text-primary`/`--bg-primary`，引用框左侧加 3px 当前色竖线，出处用 Mono 小字。

---

### Content / 内容页 — 图片网格型

出现频次：1 页

**结构**：页眉 + 页码。主体为引导小字 → 标题 → 3×2 或 3×1 图片网格。每个图片统一固定高度，使用标准比例，下方带 Mono 小字说明。

**背景**：`--bg-primary` 纯色。

**占位符**：`引导小字`、`标题`、`图片 1 说明`（共 3–6 组）

**文字颜色**：图片说明用 `--text-secondary`。

---

### Content / 内容页 — 流程步骤型

出现频次：0–1 页

**结构**：页眉 + 页码。主体为引导小字 → 标题 → 一个或多个步骤组。每个组含组标题与横向排列的步骤卡片，步骤结构为编号 → 标题 → 描述。单行建议 ≤5 个步骤。

**背景**：`--bg-primary` 纯色。

**占位符**：`引导小字`、`标题`、`组标题`、`步骤编号`、`步骤标题`、`步骤描述`

**动效**：可采用逐步点亮动效，翻到此页后按交互逐个点亮，全部点亮后才可翻页。

---

### Content / 内容页 — 并列对比型

出现频次：0–1 页

**结构**：页眉 + 页码。主体为引导小字 → 标题 → 1:1 左右双栏。左栏为「旧/错误」，透明度 0.55 做弱化；右栏为「新/正确」，满亮度突出。两栏均使用左侧 3px 当前色竖线做引用块感。

**背景**：`--bg-primary` 纯色。

**占位符**：`引导小字`、`标题`、`左栏标签`、`左栏标题`、`左栏要点`、`右栏标签`、`右栏标题`、`右栏要点`

**文字颜色**：弱化列用 `--text-secondary` 55% 透明度，强调列用 `--text-primary`。

---

### Content / 内容页 — 图文混排型

出现频次：1–2 页

**结构**：页眉 + 页码。主体为 8:4 两栏网格，左栏为正文主导：引导小字 → 标题 → lead 段 → 正文段 → 引用框；右栏为竖版或方形辅助图（3:4 / 1:1）。

**背景**：`--bg-primary` 或深色背景，light 与 dark 交替。

**占位符**：`引导小字`、`标题`、`lead`、`正文`、`引用框`、`图片说明`

**文字颜色**：正文层级分明，标题最重，正文用 `--text-secondary`，引用框左侧带当前色竖线。

---

### Statement / 金句页

出现频次：0–1 页

**结构**：页眉 + 页码。中央垂直居中：引导小字 → 超大衬线引用（分 2 行）→ 英文原文或中文释义 → 出处元数据行。

**背景**：深色背景优先，偶用浅色背景。

**占位符**：`引导小字`、`引用文字`、`释义/原文`、`出处`、`日期`

**文字颜色**：引用用 `--bg-primary` 衬线加粗，释义用 `--bg-primary` 65% 透明度。

---

### Closing / 结尾页

出现频次：1 页

**结构**：页眉 + 页码。中央垂直居中：引导小字 → 大号结语标题（可分 2–3 行断行）→ 一行点破文字。底部页脚保留结束标记。

**背景**：深色背景（hero dark）。

**占位符**：`引导小字`、`结语标题`、`点破文字`、`页脚结束标记`

**文字颜色**：标题用 `--bg-primary`，点破文字用 `--bg-primary` 70% 透明度。

## 5. Signature Elements — 标志性元素

### 杂志页眉与页脚

每页顶部与底部必须保留杂志式元数据条。顶部左侧为稳定栏目名，右侧为页码；底部左侧为中文说明，右侧为英文幕标记。两者均使用 IBM Plex Mono，全大写，字间距拉宽。

### 标题上方引导小字

标题上方的一行 Mono 小字，每页不同，作为本页钩子。示例：「5 STEPS」「THREE RULES」「Quote · 金句」。禁止与页眉栏目名重复。

### 左侧竖线引用框

左栏带 3px 当前色竖线的引用块，用于金句、关键判断或他人引言。结构：引用文字 + 底部 Mono 出处。

### 顶部横线数据卡片

三段式结构：顶部 Mono 英文标签 → 巨型数字 → 底部中文注释。卡片顶部带 2px 当前色横线。

### 标准比例图片框

所有图片必须包在图片框中，使用标准比例（16:10 / 4:3 / 3:4 / 1:1）。网格中的图片必须统一固定高度，避免使用原图奇葩比例。信息图/截图应使用 contain 填充以避免文字被裁切。

### 巨型背景字

可选装饰：极低的透明度文字，用于 hero 页或金句页。内容多为英文关键词、章节序号或「BUT/NOW/HERE」。使用时其他内容需浮于其上层。

### 行内荧光标记

行内短语的「荧光笔」效果，仅包裹 1–3 个关键词。深色主题用亮条，浅色主题用暗条。
