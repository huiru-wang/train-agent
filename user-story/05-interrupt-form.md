# P0-05 interrupt 表单信息收集流程

## 用户故事

作为培训负责人，当 AI 需要更多信息才能生成高质量 PPT 时，我希望通过一个结构化表单快速填写关键信息（如培训受众、时长、重点模块），而不是通过多轮自由对话反复澄清。

## 场景描述

Agent 执行到需要澄清信息的节点时，主动中断并向前端发送 interrupt 信号，前端渲染 clarify 表单。用户填写完毕提交后，Agent 从中断点恢复并继续生成。

## 前置条件

- 用户已触发 /ppt 命令，Agent 正在执行生成流程
- Agent graph 配置了 clarify_form 中断节点

## 核心步骤

1. Agent 判断需要更多信息，触发 interrupt（包含 title、description、fields）
2. 聊天面板渲染 clarify 表单，包含结构化字段（文本/下拉/多选）
3. 用户填写各字段（如：受众=新员工、时长=30分钟、语气=正式）
4. 用户点击提交，前端调用 resume 接口将表单数据发回 Agent
5. Agent 从中断点恢复，将表单数据注入上下文，继续生成 PPT

## 验收标准

- [ ] 表单在对话流中正确渲染，字段类型（text / select / multiselect）均正常
- [ ] 必填字段未填时，禁止提交并给出提示
- [ ] 提交后表单立即收起（本地 submitted 标记），防止重复提交
- [ ] Agent 成功接收 resume 数据并继续执行（后续工具调用卡片继续出现）
- [ ] resume 后消息列表刷新，显示完整的对话历史

## 异常场景

- resume 接口失败时，表单不收起，用户可重新提交
- 流式连接中断后重新连接，已提交的中断不再重复展示表单
