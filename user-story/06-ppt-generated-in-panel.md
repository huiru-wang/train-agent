# P0-06 PPT 生成完成并出现在产出面板

## 用户故事

作为培训负责人，我希望 PPT 生成完成后能立刻在右侧产出面板看到结果，包括文件名、类型和状态，不需要手动刷新页面。

## 场景描述

Agent 调用 save_output 工具保存生成的 HTML PPT 文件后，产出面板（TaskPanel）定时轮询到新任务记录，自动展示文件名和状态，工具调用卡片也显示保存的文件名。

## 前置条件

- Agent 已完成 PPT 内容生成
- save_output 工具调用成功，文件写入文件存储

## 核心步骤

1. Agent 调用 `save_output`（type=ppt, title=xxx, content=完整HTML）
2. 后端将文件保存至文件存储，创建 task 记录（status=completed）
3. 聊天面板中 save_output 工具卡片显示 `✓ 保存产出 文件名.html`
4. 产出面板轮询（每 5 秒）到新 task，自动渲染新卡片
5. 产出卡片展示：类型图标 + 标题 + "PPT · 已完成" + 下载按钮

## 验收标准

- [ ] save_output 工具卡片完成后显示正确的文件名（如 `新员工入职培训.html`）
- [ ] 产出面板在 5 秒内自动出现新任务卡片，无需手动刷新
- [ ] 任务卡片状态为"已完成"，显示绿色勾图标
- [ ] 文件名从 title + type 正确推导（title 中空格替换为下划线）
- [ ] 生成中（generating）时卡片显示加载动画

## 异常场景

- save_output 写文件失败时，task 状态为"失败"，卡片显示红色图标
- 轮询期间网络异常，面板不崩溃，下次轮询恢复正常显示
