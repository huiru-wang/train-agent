# P0-01 创建工作区并进入

## 用户故事

作为一名培训负责人，我希望能快速创建一个专属工作区，以便在独立的空间内管理培训文档和生成培训材料。

## 场景描述

用户首次访问产品首页，看到工作区列表（初始为空），点击"新建工作区"，填写名称后确认，系统创建成功并自动跳转到该工作区的主界面。

## 前置条件

- 用户已打开产品首页
- 后端服务正常运行

## 核心步骤

1. 用户在首页点击"新建工作区"按钮
2. 输入工作区名称（如"新员工入职培训"）
3. 点击确认创建
4. 系统返回新工作区 ID，页面跳转至 `/workspace/{id}`
5. 工作区主界面加载完成，文档面板、聊天面板、产出面板均可见

## 验收标准

- [ ] 工作区创建后出现在首页列表中
- [ ] 跳转后 URL 包含正确的工作区 ID
- [ ] 三个面板（文档/聊天/产出）正常渲染，无报错
- [ ] 工作区名称在页面标题或导航中正确显示

## 异常场景

- 名称为空时，提示用户填写名称，不允许提交
- 网络异常时，展示创建失败提示，不跳转
