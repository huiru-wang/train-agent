# P0-09 文档上传失败后的错误提示

## 用户故事

作为培训负责人，当文档上传或处理失败时，我希望看到清晰的错误原因，知道下一步该怎么做，而不是面对一个卡住的加载状态或无意义的错误码。

## 场景描述

用户上传了不支持的文件格式、文件过大，或后端解析过程中发生异常，前端能及时捕获错误并以友好的文字告知用户，文档列表中该文件标记为失败状态。

## 前置条件

- 用户尝试上传文件（可能是不支持格式、过大、或后端异常）

## 核心步骤

**场景 A - 不支持的文件格式：**
1. 用户选择 `.exe` 或 `.zip` 文件上传
2. 前端在提交前校验文件格式
3. 立即弹出提示："不支持该文件格式，请上传 PDF、DOCX 或 MD 文件"
4. 文件不进入上传队列

**场景 B - 后端解析失败：**
1. 用户上传合法格式但内容损坏的文件
2. 文件上传成功，进入处理状态
3. 后端解析失败，task 状态更新为 failed
4. 前端轮询到失败状态，文档卡片显示"处理失败"红色状态
5. 展示失败原因（如"文件内容无法解析，请确认文件未损坏"）

## 验收标准

- [ ] 不支持格式在前端拦截，给出明确格式列表提示
- [ ] 后端失败状态正确同步到前端，文档卡片变为失败态
- [ ] 失败状态下用户可删除该文档并重新上传
- [ ] 任何情况下不出现无限 loading 状态（有超时或明确失败判断）

## 异常场景

- 网络中断导致上传请求失败，提示"网络异常，请检查网络后重试"
- 并发上传多个文件时，部分失败不影响其他文件的处理
