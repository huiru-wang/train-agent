# P0-07 下载产出文件

## 用户故事

作为培训负责人，我希望能一键下载生成好的 PPT 或报告文件到本地，方便在培训现场使用或分发给学员。

## 场景描述

用户在产出面板找到已完成的任务卡片，点击下载按钮，浏览器触发文件下载，文件以正确的文件名保存到本地。

## 前置条件

- 产出面板中有状态为"已完成"的任务
- 后端文件存储中对应文件存在

## 核心步骤

1. 用户在产出面板找到目标文件卡片（状态：已完成）
2. hover 卡片，出现下载按钮（下载图标）
3. 用户点击下载按钮
4. 前端构造下载 URL：`{API_BASE}/api/files/{file_path}`
5. 通过隐藏 `<a>` 标签触发浏览器原生下载
6. 文件以 `result.filename`（如 `新员工入职培训.html`）为名保存到本地

## 验收标准

- [ ] 仅"已完成"状态的卡片显示下载按钮，生成中/失败的卡片不显示
- [ ] 下载的文件名与产出面板显示的文件名一致
- [ ] 下载的文件内容完整，HTML PPT 可在浏览器直接打开预览
- [ ] 下载按钮仅在 hover 时显示，不占用常驻空间

## 异常场景

- 后端文件已被删除时，下载返回 404，浏览器给出网络错误提示
- API_BASE 未配置时，下载 URL 回退到 `http://localhost:8000`
