# P0-02 上传培训文档并完成知识化

## 用户故事

作为培训负责人，我希望将已有的培训文档上传到工作区，让系统自动解析内容并建立知识库，以便 AI 后续能基于这些材料回答问题和生成产出。

## 场景描述

用户在工作区的文档面板中选择本地文件上传，系统后台自动完成文件解析、分块、向量化和摘要生成，最终文档出现在文档列表中，状态变为"已就绪"。

## 前置条件

- 用户已进入某个工作区
- 本地有可上传的培训文档（PDF / Word / Markdown 等）

## 核心步骤

1. 用户在文档面板点击"上传文档"
2. 从本地选择文件（支持 PDF、DOCX、MD 等格式）
3. 系统显示上传进度
4. 后端完成：文件存储 → 文本解析 → 分块 → 向量化写入 ChromaDB → 摘要生成写入 SQLite
5. 文档列表中该文件状态更新为"已就绪"，显示文档摘要

## 验收标准

- [ ] 文件上传后出现在文档列表，状态流转可见（上传中 → 处理中 → 已就绪）
- [ ] 文档摘要在列表中正确展示
- [ ] 向量数据写入 ChromaDB，可被后续 RAG 检索命中
- [ ] 支持在同一工作区上传多份文档，互不干扰

## 异常场景

- 上传不支持的文件格式时，提示支持的格式列表
- 解析失败时，文档状态显示"处理失败"并给出原因
- 文件过大时，给出大小限制提示
