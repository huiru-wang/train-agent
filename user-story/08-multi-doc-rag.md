# P0-08 多文档知识库下的精准 RAG

## 用户故事

作为培训负责人，我的工作区中可能上传了多份不同主题的培训文档，我希望 AI 在回答时能从正确的文档中检索信息，引用来源清晰，不同文档的内容不会混淆。

## 场景描述

工作区包含多份文档（如"消防安全手册.pdf"和"操作规程培训.docx"），用户提问时，rag_search 工具按语义相关性命中正确文档的片段，回答中引用角标指向对应文档，不发生跨文档混淆。

## 前置条件

- 工作区已上传 2 份及以上不同主题的文档，均为"已就绪"状态
- 各文档的向量已写入 ChromaDB，且带有正确的 workspace_id 过滤

## 核心步骤

1. 用户提问特定主题的问题（如"灭火器检查周期是多久？"）
2. Agent 调用 `rag_search`，ChromaDB 按 workspace_id 过滤后按语义检索
3. 命中消防安全相关片段，未混入操作规程文档内容
4. Agent 生成回答，引用角标指向"消防安全手册.pdf"
5. 用户 hover 引用角标，确认来源文档名正确

## 验收标准

- [ ] 多文档工作区中，RAG 检索结果来源文档与问题语义匹配
- [ ] 引用角标指向的文档名准确，不出现跨文档错误归因
- [ ] workspace_id 过滤生效，不检索其他工作区的文档内容
- [ ] 多份文档摘要均注入 Agent system prompt，Agent 知晓所有文档的存在

## 异常场景

- 问题横跨多份文档时，Agent 综合引用多个来源，分别标注角标
- 某文档向量化异常时，Agent 基于其他文档作答并说明部分文档不可用
